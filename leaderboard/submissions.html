Submit a Run

Package your agent's output and share it with the community. ← Back to leaderboard

Prerequisites

Before submitting, ensure your run meets these requirements:

  • Your agent received only the seed prompt from harnesses/<name>/prompt.md
  • No additional context, examples, or partial implementations were provided
  • The harness version matches a published, tagged release
  • No human edits were made to the agent's output

Runs that don't meet reproducibility criteria can be submitted as "unofficial."

Option A: Automated Runner

Use the built-in runner to generate a submission automatically:

pip install -e .
benchmark run --harness mini-git --agent claude-code --model claude-sonnet-4-6

This creates a submission directory in submissions/ and prints the path. Then score it:

benchmark score --submission submissions/mini-git-claude-sonnet-4-6-20260310T140000Z --harness mini-git

Option B: Manual Submission

For runs from other agents (Cursor, Devin, Copilot, etc.), create the following structure:

submissions/
  mini-git-<model>-<date>/
    metadata.json        ← Required
    workspace/           ← Agent output files
      mini_git.py
      test_mini_git.py
      ...

metadata.json Schema

{
  "model": "gpt-4o-2024-11-20",
  "agent_framework": "cursor",
  "agent_framework_version": "0.43.0",
  "scaffolding_config": {},
  "date": "2026-03-10T14:23:00Z",
  "harness": "mini-git",
  "harness_version": "1.0.0",
  "wall_clock_seconds": 847,
  "tokens_input": 45000,
  "tokens_output": 12000,
  "cost_usd": 4.23,
  "notes": ""
}
FieldRequiredDescription
modelrequiredFull model identifier including version
agent_frameworkrequiredTool used (claude-code, cursor, devin, aider, raw-api…)
daterequiredISO 8601 timestamp of when the run was performed
harnessrequiredHarness name (e.g. mini-git)
harness_versionrequiredSemver of the harness (e.g. 1.0.0)
agent_framework_versionoptionalVersion of the agent framework
wall_clock_secondsoptionalTotal elapsed wall-clock time
tokens_input / tokens_outputoptionalToken counts
cost_usdoptionalTotal cost in USD
notesoptionalFree-form context about the run

Scoring Your Submission

python -m scorer.scorecard --submission submissions/my-run/ --harness mini-git --output scorecard.json

Adding to the Leaderboard

To add your run to the public leaderboard:

  1. Score your submission to get scorecard.json
  2. Open a pull request or issue on the repository
  3. Include your metadata.json and scorecard.json
  4. A maintainer will verify and add it to leaderboard/data/runs.json
Model providers are encouraged to submit official runs. Leaderboard data is public — all runs are visible to everyone. The private held-out test layer is never published and is re-evaluated by maintainers for new submissions.
